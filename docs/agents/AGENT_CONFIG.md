## 개요

OnChain Korea는 8개 부서 에이전트 시스템으로 운영됩니다. 각 에이전트는 특정 도메인을 담당하며, 주 1회 동기화와 실시간 Slack 커뮤니케이션을 통해 협력합니다.

### 프로젝트 컨텍스트
- **제품**: GREED Academy — 8주 블록체인 교육 커리큘럼 (한국 시장 최적화)
- **현재 단계**: Phase 1 운영 (Semester 1, 4주→8주 확장) / Phase 2 준비 (DeFi 심화, 6월~8월)
- **목표**: 500+ 한국어권 학습자 모집, 40%+ 완료율
- **기술**: React 18 + Vite + Tailwind + Supabase + Vercel
- **런칭**: 2026년 5월 (Phase 1 본 과정)

## 동기화 일정

### 일일 (Daily)
- 💬 Slack 채널별 모니터링 (실시간)
- 📊 기술개발: 빌드 상태, 에러 로그
- 📊 운영생산: enrollment, 커뮤니티 활동

### 주 1회 (Weekly)
- **월요일 9:30 AM** — 경영관리 + 인사총무 (전 팀 스탠드업)
  - 주간 마일스톤 확인
  - 이슈 트래킹
  - 의사결정 필요 사항 검토

- **수요일 2:00 PM** — 영업마케팅 (수강생 모집 현황)
  - 등록율 분석
  - SNS 캠페인 성과
  - A/B 테스트 결과

- **수요일 1:00 PM** — 08_기타 (디자인 리뷰)
  - 새 컴포넌트 검토
  - 모바일 UX 테스트
  - 브랜드 일관성 확인

- **목요일/금요일 3:00 PM** — 기술개발 (코드 리뷰)
  - 배포 전 검토
  - 성능 최적화
  - 보안 업데이트

- **금요일 3:00 PM** — 운영생산 (진도 리포트)
  - 주차별 완료율
  - 수강생 피드백 요약
  - 다음 주 예측

### 월 1회 (Monthly)
- **첫째 주 금요일 4:00 PM** — 재무회계
  - 예산 현황 리뷰
  - 비용 대비 수익 분석

- **첫째 주 목요일 2:00 PM** — 법무컴플라이언스
  - 규제 동향 검토
  - 계약 법무 진행 상황

- **월 중 1회** — 영업마케팅 + 경영관리
  - 팩트체크 업데이트
  - 파트너십 협의 진행

### 분기 1회 (Quarterly)
- OKR 검토 및 업데이트
- 장기 전략 검토 (Phase 3)

## FAQ

**Q: 에이전트가 병가/휴가를 가면?**
A: 담당자와 경영관리 에이전트가 협의하여 임시 대체 에이전트를 지정합니다. 최소 24시간 전 통보.

**Q: 긴급 배포가 필요하면?**
A: #onchain-korea-emergency 채널에 공지 후 기술개발 에이전트가 즉시 대응합니다. 배포 후 회의에서 사후 검토.

**Q: 의사결정에 충돌이 있으면?**
A: 경영관리 에이전트(MinGoo)가 중재합니다. 48시간 내 결정.

**Q: 문서를 어떻게 편집하는가?**
A: 각 부서의 README.md와 주요 문서는 관련 에이전트가 유지관리합니다. Git으로 관리되며, 변경사항은 changelog.md에 기록됩니다.

**Q: Phase 변경이 발생하면?**
A: 경영관리 에이전트가 decisions.md에 기록하고, 전사 Slack #onchain-korea-management에서 공지합니다. 모든 부서는 영향받는 범위를 24시간 내 자신의 README.md에 반영합니다.
