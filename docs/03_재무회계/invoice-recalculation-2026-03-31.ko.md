# OnChain Korea 인보이스 재산정 메모

작성일: 2026-03-31  
기준: 기존 4주 인보이스의 업데이트 메모  
주의: 현재 대화에 첨부된 인보이스 이미지는 텍스트 판독이 불가능하여, 이 문서는 "변경 요청된 항목"을 기준으로 재산정한 기준안입니다. 기존 인보이스의 나머지 항목과 총액은 읽을 수 있는 원본이 들어오면 그대로 덮어써야 합니다.

## 이번에 반영한 변경점
- `cNFT 민팅 비용`은 최신 Metaplex Bubblegum V2 기준으로 다시 계산
- `담당인력 2인 1조 상시 모니터링`은 8주 운영 기준으로 상향 반영

## 1. cNFT 민팅 비용 재산정

### 산정 기준
- 목표 수량: `500개`
- 프로토콜: `Metaplex Bubblegum V2`
- 트리 구성: Metaplex CLI 권장 소형 트리
  - `16,384` capacity
  - `maxDepth 14`
  - `maxBufferSize 64`
  - `canopyDepth 8`
- 내부 예산 환산용 보수 기준: `1 SOL = ₩200,000`
  - 실제 시장 가격이 아니라, 내부 견적용 완충 환산치

### 공식 기준 수치
- Merkle Tree 생성 비용: `~0.34 SOL`
- Bubblegum V2 Create 프로토콜 수수료: `0.00009 SOL / 1개`

### 계산
- 트리 생성 1회: `0.34 SOL`
- 500개 민팅 프로토콜 수수료: `500 x 0.00009 SOL = 0.045 SOL`
- 소계: `0.385 SOL`
- 트랜잭션 수수료 및 실행 여유 버퍼 포함 예산값: `0.50 SOL`

### 인보이스 반영 권장 금액
- 권장 반영 금액: `₩100,000`
- 실무 반영 문구:
  - `cNFT Minting Network Reserve (500 units): ₩100,000`

### 메모
- 온체인 순수 네트워크 비용은 매우 작습니다.
- 인보이스에서 금액을 더 크게 잡아야 한다면, 그것은 `민팅 운영 실행`, `메타데이터 QA`, `지갑/배포 확인`, `발급 실패 대응` 같은 운영성 항목으로 분리하는 것이 맞습니다.
- 순수 체인 비용만 놓고 보면 `₩100,000`이면 충분히 보수적입니다.

## 2. 담당인력 2인 1조 상시 모니터링 비용 상향

### 산정 기준
- 운영 기간: `8주`
- 인력 구성: `2인 1조`
- 운영 방식: 상시 모니터링 / 문의 대응 / 이슈 에스컬레이션 / 발급 이슈 확인 / 학습 진행 모니터링
- 내부 보수 단가: `1인 1일 ₩300,000`
  - 단순 CS가 아니라, 온체인 이슈 응대와 운영 모니터링이 포함된 로디드 단가로 계산
- 근무일 기준: `주 5일 x 8주 = 40일`

### 계산
- `2명 x 40일 x ₩300,000 = ₩24,000,000`

### 인보이스 반영 권장 금액
- 권장 반영 금액: `₩24,000,000`
- 실무 반영 문구:
  - `Dedicated Monitoring & Operations (2-person team, 8 weeks): ₩24,000,000`

### 메모
- 이 금액은 "높게 잡아야 한다"는 요청에 맞춰 보수적으로 반영한 값입니다.
- 만약 실제 운영을 주말/야간 대응까지 포함한 사실상 24/7 체계로 명시하려면, 이 항목은 `₩28,000,000 ~ ₩32,000,000` 구간까지도 올릴 수 있습니다.
- 현재 기준안은 과도하게 부풀리지는 않되, 낮지도 않은 `₩24,000,000`으로 고정합니다.

## 권장 후속 조치
- 읽을 수 있는 원본 인보이스를 다시 받으면:
  - 기존 모든 라인 아이템을 동일 순서로 복원
  - 위 2개 항목만 교체
  - 4주 기준 문구를 `8주 기준`으로 전면 수정
  - 최종 총액과 VAT 포함 합계를 다시 계산
